# Issue 287/288 README and KDoc sync

## Context

Issues #287 and #288 were the remaining small documentation gaps in
`bluetape4k-leader`: root Korean README content lagged management/observability
docs, and `LeaderElectionManagementRegistry` did not meet the public KDoc
Behavior/Contract standard.

## Decision

Keep the root `README.md` and `README.ko.md` synchronized by adding both the
Spring Boot Actuator endpoint and Ktor management route sections to both
locales. Expand only the public KDoc for `LeaderElectionManagementRegistry`;
do not change behavior.

## Outcome

The root READMEs now document `GET /actuator/leaderElection`,
`bluetape4k.leader.observability.lock-names`, and
`GET /management/leaderElection`. The Ktor registry KDoc now states thread
safety, deterministic snapshots, JVM-local scope, and basic usage.
