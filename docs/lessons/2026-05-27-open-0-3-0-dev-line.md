## Context

`bluetape4k-leader` 0.2.2 was published and verified on Maven Central. The
next open milestone is 0.3.0 with feature/design work, while the 0.2.2
milestone has no open issues.

## Decision

Move `baseVersion` to `0.3.0` and keep `snapshotVersion=` empty in git. The
snapshot publish workflow remains responsible for injecting
`-PsnapshotVersion=-SNAPSHOT`.

## Outcome

`develop` now resolves as the next 0.3.0 development line locally, while stable
release tags still require an empty `snapshotVersion` and an exact
`baseVersion` match.

## Verification

Run a lightweight Gradle properties check plus `git diff --check`; no README
stable coordinate changes are needed because 0.2.2 remains the latest published
release.

## Future Guidance

After every stable release, advance `baseVersion` to the next planned milestone
and keep `snapshotVersion=` empty unless the publish workflow explicitly
overrides it.
