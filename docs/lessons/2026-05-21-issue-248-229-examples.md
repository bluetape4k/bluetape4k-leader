# Issue 248/229 examples

## Context

Milestone 0.1.1 needed two adoption examples:

- Kubernetes Lease leader election using `K3sServer.Launcher.k3s`.
- Redis leader election combined with distributed Bucket4j rate limiting.

## Decision

Keep both as non-published examples:

- `examples/k8s-lease` owns the Fabric8 typed Lease walkthrough and isolates the
  K3s container test in a dedicated `k8sTest` Gradle task tagged `k8s`.
- `examples/rate-limiter` reuses the existing Redis `LettuceLeaderElector`
  pattern for single-node dispatch and uses `DistributedSuspendRateLimiter`
  over a Lettuce Bucket4j proxy for aggregate API quotas.

## Outcome

The examples are registered in Gradle settings, CI path filters, normal CI test
jobs, and the scheduled/manual Examples workflow. Normal CI runs the lightweight
`k8s-lease:test` lane; the privileged K3s acceptance test runs via
`examples:k8s-lease:k8sTest`.

## Verification

- `actionlint .github/workflows/ci.yml .github/workflows/examples.yml`
- `./gradlew :examples:k8s-lease:test :examples:rate-limiter:test --no-daemon`
- `./gradlew :examples:k8s-lease:k8sTest --no-daemon`
- `./gradlew build -x test -x koverVerify --parallel --no-daemon`

IDE diagnostics were checked for the touched Kotlin example files after the
worktree became available to IntelliJ MCP. The files reported `problemCount=0`
and no build errors; file analysis was not fresh because the files were not open
in the editor, so Gradle compile and tests remain the primary verification.

## Future Agents

When adding more example modules, update `settings.gradle.kts`,
`.github/workflows/ci.yml`, `.github/workflows/examples.yml`, README locale
pairs, and a short lesson in one pass. Keep privileged container tests out of the
default CI path unless the runner contract explicitly supports them.
