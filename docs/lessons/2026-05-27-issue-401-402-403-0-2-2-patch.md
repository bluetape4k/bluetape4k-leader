# Leader 0.2.2 patch issues

## Context

Milestone `0.2.2` had three patch issues:

- #401: Mongo group electors accepted a history recorder but did not emit group lifecycle events.
- #402: AOP Micrometer metrics counted successful acquisition but did not time acquisition latency.
- #403: group `FAIL_OPEN_RUN` did not treat an existing fail-open sentinel as a reentrant scope.

## Decision

Wire Mongo group history through sync, async, and suspend paths, using the backend lock token as
the audit token and the group slot as `slotId`. Record terminal history events before unlock while
keeping cancellation semantics unchanged.

Add `leader.aop.acquire.duration` as a separate timer on successful AOP acquisition, and document it
in both README locales.

Short-circuit matching group fail-open sentinels before backend acquisition in the sync AOP branch.

## Outcome

The patch closes #401, #402, and #403 without adding dependencies or changing public API shape.
