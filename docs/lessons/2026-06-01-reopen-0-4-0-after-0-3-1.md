# Reopen 0.4.0 after leader 0.3.1

## Context

The `0.3.1` patch release temporarily moved `baseVersion` back from `0.4.0` to
`0.3.1` so the tag-triggered release workflow could verify the tag version
against `gradle.properties`.

## Decision

Reopen the `0.4.0` development line immediately after the `0.3.1` tag and
release workflow so new work does not accidentally publish another 0.3.x
artifact.

## Outcome

Public README coordinates remain on the latest stable `0.3.1`, while the build
line returns to `baseVersion=0.4.0`.

## Verification

- `0.3.1` tag was pushed from the release-prep commit.
- `0.3.1` GitHub Release was created by the release workflow.

## Future Guidance

For patch releases made from an already-opened minor development line, always
restore the development `baseVersion` after tagging the patch release.
