# Open 0.4.0 Development Line

## Context

`bluetape4k-leader` 0.3.0 was published through the release workflow and
verified on Maven Central for the BOM, `leader-core`, and
`leader-redis-redisson` POMs.

## Decision

Move `develop` to the next minor line with `baseVersion=0.4.0` and keep
`snapshotVersion=` empty. Non-bugfix work should continue to be grouped into
minor releases rather than patch releases.

## Outcome

Local Gradle version resolution now points at the next development line while
README dependency coordinates continue to advertise the latest stable release,
0.3.0.

## Verification

- GitHub release workflow `26727897102` completed successfully.
- Maven Central returned HTTP 200 for the 0.3.0 BOM, core, and Redisson POMs.
- Run `./gradlew properties --no-daemon | rg '^version:|^group:'` after the
  version bump.
